Highlighted features

Making it simple

Unfortunately, microservices isn't a free lunch. The overhead of maintaining and operating your service scales with the number of services. Therefore mikro.io works hard at lowering that overhead. Actually we're doing everything we can to make you faster.

Deploying a change is just a click (or a commit) away. We support the most common patterns such as canary releases and blue/green deployment, and integrate with tools like CircleCI.

Setting up a new service takes literally 30 seconds.

We like octocats too.

We think Github is a great tool for collaboration and developer productivity. We hope you think so too. Therefore have deeply integrated Github into mikro.io to get that tight connection between your code and your production environment.

Every project on mikro.io is attached to a Github organization. Members of the organizatin automatically gets access to mikro.io. No need to manage teams and members in multiple places.

You probably have more important things to do anyway.

Know what's running

It's kinda fundamental that you wanna know what's running in your cluster.

mikro.io gives you an overview of all the tasks for your service and their state. You can drill down and see why they stopped and filter through the logs.

How it works

1

You give us access to your AWS account. We'll use it carefully, we promise.

2

We create a Elastic Container Service cluster and an autoscaling group that we'll manage for you. This is where your services will run.

3

Create a service and connect it your Github repository. Build an container image, push it and deploy. It's that simple.
